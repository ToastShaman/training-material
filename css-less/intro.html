CSS

  • This is an li tag
  • In HTML ids are used to represent a unique element
  • classes are used to represent several elements that have some semantically similar property
  • CSS allows us to target elements using a 'selector' and apply styles to the targeted elements
  • Selectors can consist of tag names, ids or classes most of the time and can be combined

  • Inheritance
  • Specificity has four components; let’s call them a, b, c and d. Component “a” is the most distinguishing, “d” the least.
    • Component “a” is quite simple: it’s 1 for a declaration in a style attribute, otherwise it’s 0.
    • Component “b” is the number of id selectors in the selector (those that begin with a #).
    • Component “c” is the number of attribute selectors—including class selectors—and pseudo-classes.
    • Component “d” is the number of element types and pseudo-elements in the selector.
  • Source order - if specificity is the same, the last rule wins

LESS

  • LESS is a superset of CSS ie LESS is CSS and more
  • We can write any standard CSS in a .less file
  • LESS is powerful because it allows us to nest CSS rules, use variables and write functions for repeated CSS which keeps our CSS DRY, easier to maintain and read
  • Nested CSS
  • Variables
  • Mixins
  • Mixins can be parametrised
  • We can import other less files, once the less gets compiled the normal CSS rules about inheritance, specificity and source order apply

Boot strap

  • Grids are made up of rows
  • Rows are divided into 12 columns
  • You can group columns together to make wider columns
  • We often want to optimise our site to work well on devices with smaller screens
  • Bootstrap allows us to specify different sized columns for different size screens (eXtra-Small, SMall, MeDium, Large
  • col-xs-* col-sm-* col-md-* col-lg-*, Each class scales up, so if we want to set the column width for all screens we only need to set the col-xs-* class
